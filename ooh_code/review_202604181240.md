## 2. Summary / 总结
这篇论文研究 many-to-one DRT 中“家门口接送 vs. meeting point”与动态定价联动的问题，选题有现实意义，建模、附录和实验组织也明显比一般应用型论文更完整。文稿最强的部分是问题定义清晰、实验面比较宽、作者也主动收缩了部分结论；但当前版本的核心短板在于若干 headline claims 仍然强于证据本身，尤其是“改进来自 SPO 本身”这一点并没有被现有消融结果充分支撑，同时理论近似与行为模型校准的外部有效性仍偏弱。

## 3. Strengths / 优点（按重要性排序）
- 论文问题设定清楚且有实际价值：把 sequential arrivals、home pickup、meeting points、outside option 和 dynamic pricing 放在同一 many-to-one DRT 框架里，应用场景明确。
- 方法链条完整：从 MDP 表述、候选点生成、状态编码、边际成本预测到 pricing oracle 与 SPO+ 训练，整体 pipeline 是自洽的。
- 实验覆盖面较好：包含六个 synthetic baseline、DSPO 对比、Beijing corridor case、fleet-size robustness、one-at-a-time sensitivity 和 joint sensitivity，信息量充足。
- 作者对局限性的披露是诚实的，尤其在 Beijing case、synthetic behavior calibration、clipped pricing heuristic 和 proxy interpretation 上，已经开始主动降调。
- 写作可读性较好，appendix 对符号、Bellman-to-pricing 推导和 pricing structure 的补充提升了 self-containedness。

## 4. Weaknesses / 缺点（按严重程度排序）
- **CRITICAL**: 论文对 “SPO 本身带来稳定且可测的改进” 的 headline 表述仍然超过了证据强度。`paper/trc_latex/manuscript.tex:105,109,160,926,1030` 多处把增益明确归因于 SPO；但真正隔离了 loss effect 的 ablation 在 `:852` 只给出 `0.28%` 的平均净利润增益、`19/30` seed 胜出，并且作者自己也写了 “effect size is small and should be interpreted as directional evidence”。这意味着当前证据更支持“DRPO 这套完整 pipeline 优于 DSPO baseline”，而不是强支持“核心提升来自 SPO+ loss”。
- **MAJOR**: 理论叙事仍然比实际算法依据更“硬”。文稿一方面用 Bellman continuation value 引出 decision-focused learning，另一方面真实训练与部署却依赖两个明显 heuristic 的对象：一是用 ex-post terminal marginal cost proxy 代替 `\Delta V_k(s_t)`，见 `:505` 与 `:1188`；二是 pricing oracle 在 box constraints 下实际采用 clipped common-markup heuristic，而不是 exact constrained optimizer，见 `:1295-1303`。目前的诊断与说明能证明“heuristic 可能有用”，但还不足以完全支撑较强的理论口径。
- **MAJOR**: 行为模型的外部有效性仍然偏弱，且 MNL 在“home + 多个相近 meeting points”的选项结构下可能过于强。作者已明确说明 choice parameters 是 synthetic / literature-informed 而非实证估计，见 `:381,795,932,1030`；但论文仍把利润与 quit-rate 改善写得较接近可部署结论。对于高度相似的多个 meeting-point 备选，MNL 的 IIA 假设尤其值得讨论，否则 substitution pattern 可能被系统性简化。
- **MINOR**: Beijing case 的证据更适合作为 qualitative stress test，而不是较强的 corridor-level 经验支撑。文中已承认只有 5 seeds、区间较宽、结论是 directional evidence，见 `:754,911,924,926`，但主文一些表述仍略显积极。
- **MINOR**: reproducibility 还有提升空间。`Data and code availability` 在 `:1035` 写的是“acceptance 后公开 replication package”，这在投稿阶段会削弱可复核性；另外 predictor 第二输出在文中有时被称为 cumulative in-vehicle time proxy，有时图里接近 “pickup-position head”，见 `:431,535`，术语可再统一。

## 5. Actionable Fixes / 对每个 CRITICAL 或 MAJOR 问题的具体修改建议
1. 针对 SPO 归因过强的问题：
   将论文主 claim 明确降为“DRPO 作为 DSPO 的 DRT-adapted pipeline 在 tested settings 中优于 DSPO baseline；SPO+ 的独立贡献在当前 ablation 中是 modest / directional”。相应修改 highlights、abstract、introduction contribution paragraph 和 conclusion，把 `consistent gains over DSPO`、`SPO extension improves DSPO in a stable way` 这类表述改成更谨慎的版本。
2. 针对 SPO 归因过强的问题：
   在结果部分补一组正式的 paired statistical analysis，至少对 `DRPO vs DSPO-ablation` 报告 paired bootstrap CI 或 Wilcoxon signed-rank test、配套 effect size，并把这组结果放到主文而不是只留一个小表。现在最关键的不是再报更多平均值，而是把“loss effect 的大小到底有多大”说清楚。
3. 针对理论与算法之间的 heuristic gap：
   增加一个专门的小节，把两层近似讲透并做定量验证。第一层是 `\Delta V_k(s_t) -> c_{tk}` 的 proxy gap；第二层是 exact constrained pricing -> clipped common-markup heuristic 的 oracle gap。前者建议不只报一次 Spearman，而是跨更多 held-out episodes 报 rank correlation、top-1 agreement 或 regret proxy；后者建议在小规模实例上与 exact solver 或更强 local search 做对照，证明 heuristic oracle 不会显著扭曲下游结论。
4. 针对理论与算法之间的 heuristic gap：
   在全文中把理论口径统一改成“decision-focused heuristic with an internally aligned pricing oracle”，避免给读者造成“这里有一个 exact downstream optimization oracle”的印象。附录中的 `Approximation validity` 和 `clipped common-markup rule` 两段其实已经提供了基础，只需要把这种谨慎口径前移到主文。
5. 针对行为模型外部有效性不足：
   最有力的修复是补充一个实证校准来源，哪怕是 stated-preference calibration 或来自相近 microtransit / customized bus 文献的 transfer calibration，也会显著提升说服力。如果做不到，至少要新增 choice-model robustness：例如改变 home-vs-meeting-point 的相关性结构，或给出一个 non-MNL robustness check，检验核心结论是否依赖 IIA。
6. 针对行为模型外部有效性不足：
   进一步压缩 practical claim。建议把 Beijing case 明确写成 “semi-realistic policy illustration under literature-informed behavior assumptions”，避免读者把 1.35% 或 1.86% 这样的数值理解为真实 corridor 上可直接预期的 operational uplift。
